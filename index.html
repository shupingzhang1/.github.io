2.抽象类

2.1概述：

抽象就是看不懂的

特点：

a.抽象类和抽象方法都必须用abstract关键字修饰 abstract class 类名(); public abstract void eat{};
b.抽象类不一定有抽象方法，有抽象方法的类一定抽象类或者接口。
c.抽象类不能实例化。按照多态的方式由具体的子类实例化，抽象类多态，多态的一种。
d.抽象类的子类：要么是抽象类，要么重写抽象类中所有的抽象方法。

###2.1抽象类成员的特点：
a.成员变量：既可以是变量也可以是常量，abstract 不能修饰成员变量。
b.构造方法：用于子类访问父类数据的初始化。
c.成员方法：即可以是抽象的也可以是非抽象的。

抽象类的成员方法特性：
a.抽象方法 强制要求子类做的事情。
b. 非抽象方法 子类继承的事情，提高代码的复用性。

抽象类面试题：
1.一个 抽象类可以没有抽象方法，目的：不让其他类创建本类对象。
2.abstract不能和哪些关键字共存？
a.abstract和static 原因：被abstract修饰的方法没有方法体，被static修饰的可以用类名直接调用，但是类名调用抽象类是没有意义的。
b.abatract final
被abstract修饰的方法强制子类重写
被final修饰的方法不让子类重写
c.abstract private
private 不让子类看到
abstract 修饰的方法强制子类重写

2.3接口：

2.3.1接口概述：从狭义的角度讲的是Java中的interface，从广义的角度将对外提供规则的都是接口。

2.3.2接口特点

a.接口用关键字interface表示  interface 接口名{}
b.类实现接口用implements表示  class 类名 implements 接口名{}
c.接口不能实例化   按照多态的方式实例化
d.接口的子类 可以是抽象类，但意义不大。  可以是具体类，要重写接口中所有抽象方法（推荐）

2.3.3接口成员特点

a.成员变量：只能是常量，并且是静态的公共的,默认修饰符 public static final 三个关键字没有位置规定，手动给出
b.构造方法：接口没有构造方法
c.成员方法： 只能是抽象方法，默认修饰符 public abstract 建议手动给出

2.4关系

a.类与类关系：继承关系，只能单继承，可以多层继承
b.类与接口：实现关系，可以单实现也可以多实现，并且可以在继承一个类同时实现多个接口
c.接口与接口：继承关系，可以单继承，也可以多继承
